# Web App

Target:
- React + TypeScript
- desktop-first responsive layout
- IndexedDB for local persistence
- optional PWA support after core parity

Principles:
- preserve product behavior from Flutter
- adapt layout for large screens instead of mirroring the mobile UI
- keep notifications optional in the first release

Implementation order:
1. shared DTOs from `product-spec`
2. local persistence
3. onboarding and routing
4. local plan generation
5. plan and calendar views
6. workout logging
7. settings
8. pace calculator
9. progress and insights
10. AI enrichment
11. PWA/offline polish
